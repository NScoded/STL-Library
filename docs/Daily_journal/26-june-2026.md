# 2. Planned Redis Lite Architecture

Studied how a real Redis server works.

Instead of reading commands from the keyboard, Redis receives commands over TCP sockets.

Future architecture:

```text
Client
   │
TCP Socket
   │
Redis Lite Server
   │
Command Parser
   │
HashMap
   │
Response
   │
Client
```

The next architectural change identified is separating command execution from the user interface.

Current design:

```cpp
getline(cin, input);
```

Future design:

```cpp
execute(command);
```

The same `execute()` function will later work with both:

* CLI
* Network clients

without changing Redis logic.

# 5. socket()

Learned that:

* Creates a TCP communication endpoint.
* Returns a File Descriptor.
* Does **not** assign an IP or Port.

---

# 7. bind()

Learned that `bind()` associates:

```
Socket

↓

IP Address

+

Port
```

without which a server cannot receive incoming connections.

---

# 8. listen()

Learned that `listen()` changes the socket into a listening state so the operating system starts accepting incoming connection requests.

---

# 9. accept()

Learned that:

* Blocks until a client connects.
* Creates a **new connected socket**.
* The original socket continues listening for future clients.

---

# 10. recv() / send()

Studied how network communication actually occurs.

```
Client

↓

send()

↓

Operating System

↓

TCP Stack

↓

recv()

↓

Server
```

Responses travel back using `send()`.

---

# 11. File Descriptor

Corrected an earlier misconception.

Previously assumed FD represented a disk address.

Learned that:

A File Descriptor is simply an index into the process's file descriptor table.

It can refer to:

* File
* Socket
* Pipe
* Terminal

For sockets:

```
FD

↓

Kernel Socket Object

↓

TCP Buffers

↓

Network Interface
```

---

# 12. Ports

Learned that ports are simply numbers identifying applications.

Examples:

```
80     HTTP

443    HTTPS

3306   MySQL

6379   Redis
```

Also studied:

* Well-known ports
* Registered ports
* Ephemeral ports

Learned that custom applications can choose almost any unused port above 1024.

---

# 13. IP Addresses

Clarified an important misconception.

A computer can have multiple IP addresses simultaneously.

Examples:

* Loopback
* Wi-Fi
* Ethernet
* VPN
* Virtual Interfaces

Also understood the purpose of:

```
127.0.0.1

(Localhost)
```

and

```
INADDR_ANY
```

which allows a server to listen on every available network interface.
