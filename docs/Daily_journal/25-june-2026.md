# Work Completed

## 1. Configured CMake Build System

Created the initial **CMakeLists.txt** for the project.

The build system now:

- Detects the C++ compiler automatically.
- Uses the required C++ standard.
- Includes all header directories.
- Builds the project executable.
- Provides a clean and platform-independent compilation process.

Learned how CMake separates:

- Source files
- Header files
- Build directory
- Generated binaries

This makes the project scalable as more containers are added in the future.

## 2. Organized Project Structure

Reorganized the repository into a cleaner layout.

Current structure includes:

- `include/`
- `src/`
- `tests/`
- `docs/`
- `build/`

This separation improves maintainability and keeps implementation, headers, documentation, and testing independent.

---

## 3. Added Include Directory Configuration

Configured

```cmake
target_include_directories(...)
```

to allow source files to include project headers without writing long relative paths.

This makes header inclusion cleaner and follows common C++ project conventions.

## 4. Improved Documentation Structure

Instead of maintaining one large development log, the documentation was divided into individual daily journals.

Created separate build log files for each development day.

Benefits:

- Easier progress tracking
- Better version history
- Simpler documentation maintenance
- Cleaner repository structure

---

## 5. Learned CMake Workflow

Practiced the complete CMake workflow.

Learned how to:

- Configure a project
- Generate build files
- Compile the project
- Rebuild after code changes

Typical workflow:

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

Also understood why out-of-source builds are preferred over compiling directly inside the project directory.
